# 01. 后端领域模型与数据库设计

## 1. 设计目标

- 对齐已验证规则字段：`PE/DIR/MD5/REG/MD5_REG/IP/CtrlWnd/AntiThread/ThreadControl`
- 支持多租户代理体系、分组策略、模块分配、发布回滚、审计
- 为多区域部署保留全局唯一 ID 与发布版本能力

## 2. 领域模型（聚合）

- **租户域**
  - `tenant`（平台租户）
  - `agent`（代理商，可层级）
  - `user_account`（管理员/代理/子账号）
- **规则域**
  - `rule`（规则主表）
  - `rule_version`（规则快照）
  - `rule_publish`（发布批次）
  - `rule_publish_item`（发布到目标明细）
- **分组域**
  - `client_group`（分组）
  - `client_group_rule`（分组绑定规则）
  - `client_group_ip`（IP 到分组映射）
  - `client_group_policy`（分组策略参数）
- **模块域**
  - `module`（模块制品定义）
  - `agent_module`（代理可用模块）
  - `client_group_module`（分组启用模块）
- **客户端域**
  - `client_device`（客户端资产）
  - `client_heartbeat`（心跳）
  - `client_rule_state`（规则版本状态）
  - `client_module_state`（模块状态）
- **审计域**
  - `audit_log`（后台操作审计）
  - `event_log`（系统事件/错误）

## 3. 关键表说明

### 3.1 rule（规则主表）

核心字段：

- `id` bigint PK
- `tenant_id` bigint
- `agent_id` bigint
- `orgn_share_agent_id` bigint
- `name` varchar(80)
- `enable_ad` tinyint
- `is_share` tinyint
- `is_hide` tinyint
- `run_possibility` int
- `remark` text
- `pe_json` longtext
- `dir_json` longtext
- `md5_json` longtext
- `reg_json` longtext
- `md5_reg_json` longtext
- `ip_json` longtext
- `ctrl_wnd_json` longtext
- `anti_thread_json` longtext
- `thread_control_json` longtext
- `used_count` bigint
- `created_at` datetime(3)
- `updated_at` datetime(3)
- `deleted_at` datetime(3) nullable

唯一约束：

- `uk_rule_agent_name`：`(agent_id, name, deleted_at)`（逻辑删除后可重建）

### 3.2 rule_version（规则快照）

- 每次修改规则生成快照，支持回滚与差异审计。
- 字段同 `rule` 内容字段，附加 `version_no`、`change_type`、`operator_id`。

### 3.3 rule_publish / rule_publish_item（发布批次）

- `rule_publish`：发布主记录（草稿、灰度、完成、回滚）
- `rule_publish_item`：批次目标（agent/group/device）和结果状态

### 3.4 client_group / client_group_rule / client_group_ip

- `client_group`：支持默认分组与 `boot_bat`
- `client_group_rule`：绑定关系 + 排序权重
- `client_group_ip`：便于无盘与公网变动场景映射

### 3.5 module / agent_module / client_group_module

- `module`：制品 URL、MD5、签名摘要、运行参数模板
- `agent_module`：代理维度授权
- `client_group_module`：分组维度启用/禁用与覆盖参数

### 3.6 client_device 系列表

- `client_device`：MAC、公网 IP、主机名、OS、客户端版本、组信息
- `client_rule_state`：设备当前规则版本、拉取时间、应用状态
- `client_module_state`：模块下载、校验、加载状态
- `client_heartbeat`：在线证明与性能基础指标

### 3.7 audit_log

建议固定字段：

- `operator_id`, `role`, `action`, `resource_type`, `resource_id`
- `before_json`, `after_json`, `request_id`, `ip`, `user_agent`
- `created_at`

## 4. 数据一致性策略

- 所有关键表增加 `version`（乐观锁）字段。
- 发布采用“批次版本号 + 规则版本号”双版本。
- 客户端以 `publish_version` 为准，避免跨区域短暂不一致。

## 5. 索引建议

- `rule(agent_id, enable_ad, is_hide, updated_at)`
- `client_device(agent_id, group_id, last_seen_at)`
- `rule_publish(status, created_at)`
- `audit_log(resource_type, resource_id, created_at)`

## 6. 迁移约定

- 使用 Alembic 管理版本迁移。
- 禁止手工改库，所有变更走迁移脚本。
- 每次迁移附带回滚 SQL。
