# 最终版规则数据结构文档

## 1. 文档目的

本文档基于实测结果整理广告猎手规则的最终字段结构，作为复刻系统时的后端入库、前端表单、客户端解析统一标准。

已通过真实新增验证字段：

- `PE`
- `DIR`
- `MD5`
- `REG`
- `MD5_REG`
- `IP`
- `CtrlWnd`
- `AntiThread`
- `ThreadControl`

## 2. 规则主记录结构

建议表名：`规则主表`

核心字段如下（建议全部保留）：

- `Id`：规则ID（字符串或整型主键）
- `AgentID`：所属代理ID
- `OrgnShareAgentID`：原始共享来源代理ID
- `AdName`：规则名称（2~40字符）
- `EnableAd`：是否启用（`"1"` 启用，`"0"` 禁用）
- `IsShare`：是否共享（`"1"` / `"0"`）
- `is_hide`：是否隐藏（`"1"` / `"0"`）
- `Remark`：备注
- `run_possibility`：执行概率（通常 `100`）
- `RuleUsedCount`：使用计数
- `LastModify`：更新时间
- `AgentName`：所属代理名称（可冗余）
- `bk_from_adrule_id` / `bk_desc` / `bk_id_list` / `bk_desc_list`：备份与版本链相关字段

规则内容字段（JSON字符串）：

- `PE`
- `DIR`
- `MD5`
- `REG`
- `MD5_REG`
- `IP`
- `CtrlWnd`
- `AntiThread`
- `ThreadControl`

说明：

- 这些字段在库中通常保存为 JSON 字符串（不是关系表）。
- 允许空字符串；未配置则为空。

## 3. 各规则字段结构定义

---

### 3.1 文件类规则：`PE` / `DIR` / `MD5`

三者结构一致，都是“父进程 -> 子路径/目标 -> 权限数组”。

```json
{
  "*": {
    "\\\\某路径\\\\目标.dll": ["noCreate", "noWrite", "LevelEnable_Self"]
  }
}
```

常见权限值：

- `noRead`：禁读
- `noWrite`：禁写
- `noExecute`：禁执行
- `noOpen`：禁打开
- `noCreate`：禁创建
- `noDelete`：禁删除
- `WhitePass`：白名单
- `LevelEnable_Self`：作用于本进程
- `LevelEnable_Child`：作用于子进程

---

### 3.2 注册表类规则：`REG` / `MD5_REG`

结构同文件类规则：

```json
{
  "*": {
    "\\\\HKEY_CURRENT_USER\\\\Software\\\\XXX": ["noWrite", "noCreate", "LevelEnable_Self"]
  }
}
```

---

### 3.3 网络规则：`IP`

结构为“进程 -> 方向(in/out) -> 目标列表 -> 协议端口”。

```json
{
  "testproc.exe": {
    "out": [
      {
        "127.0.0.1": {
          "level": 1,
          "tcp": ["80", "443"]
        }
      }
    ]
  }
}
```

字段说明：

- `in` / `out`：入站/出站方向
- `level`：范围编码（由页面勾选转换）
  - `1`：本进程
  - `2`：子进程
  - `3`：本进程+子进程
- `tcp` / `udp`：端口数组（字符串数组）

---

### 3.4 窗口规则：`CtrlWnd`

结构示例：

```json
{
  "CheckProcessTime": "20000000",
  "CheckProcessTimespan": "2000",
  "CheckProcessStartDelay": "3",
  "CwRule": [
    {
      "ProcessName": "testproc.exe",
      "ProcessChildType": 11,
      "StartDelay": 1,
      "CwWindowProp": [
        {
          "ClassName": "",
          "WindowText": "广告",
          "Width": "300",
          "Height": "250",
          "clct": "0",
          "sl": "0"
        }
      ]
    }
  ]
}
```

字段说明：

- `ProcessChildType`：位掩码组合
  - `1` 本进程
  - `2` 子进程
  - `4` 不注入
  - `8` 后关闭
  - `16` 只隐藏
  - `32` 加退出
- `CwWindowProp`：窗口匹配与处理参数
  - `ClassName`：窗口类名
  - `WindowText`：窗口标题
  - `Width` / `Height`：窗口尺寸
  - `clct`：次数控制
  - `sl`：睡眠/间隔

---

### 3.5 线程注入规则：`AntiThread`

结构示例：

```json
{
  "\\\\testproc.exe": {
    "\\\\explorer.exe": ["noThread", "LevelEnable_Self"],
    "\\\\chrome.exe": ["NoOpen", "NoKill", "noThread", "LevelEnable_Child"]
  }
}
```

常见权限值：

- `NoOpen`：禁打开
- `NoKill`：禁关闭
- `NoSuspend`：禁挂起
- `NoWMOP`：禁读写
- `noThread`：禁注入
- `KillChildProc`：关子进程
- `KillProc`：关进程
- `ProtectProc`：保护
- `WhitePass`：白名单
- `LevelEnable_Self` / `LevelEnable_Child`：作用范围

---

### 3.6 线程控制扩展：`ThreadControl`

该字段在历史规则中常为空，但实测可写入并落库。建议保留为 JSON 字符串扩展字段。

示例：

```json
{
  "Enable": "1",
  "Note": "codex_verify"
}
```

## 4. 接口层提交建议

建议统一接口：

- 新增：`POST /rule_mng/insert_rule`
- 修改：`POST /rule_mng/update_rule`

`Content-Type` 推荐 `application/x-www-form-urlencoded`（兼容原系统），或新系统改为 `application/json`。

提交建议字段：

- 基础：`AdName`, `EnableAd`, `IsShare`, `Remark`
- 规则：`PE`, `DIR`, `MD5`, `REG`, `MD5_REG`, `IP`, `CtrlWnd`, `AntiThread`, `ThreadControl`

## 5. 校验规则建议（复刻时）

- 规则名称：2~40字符
- 每个规则字段最大长度：10000字符（沿用原逻辑）
- 规则字段必须是合法 JSON 字符串（空串允许）
- 至少一个规则字段非空（可配置）
- 需要做重复键检查（同级键名不重复）
- 路径统一规范化（可做自动补 `\\` 前缀）

## 6. 数据库存储建议

- 主表：`rules`
- 内容字段统一 `TEXT`/`LONGTEXT` 存 JSON 字符串
- 索引建议：
  - `(AgentID, AdName)` 组合索引
  - `EnableAd`, `IsShare`, `is_hide` 普通索引
  - `LastModify` 索引

可选增强：

- 增加 `rules_history` 做版本快照
- 增加 `rules_publish_log` 做发布审计

## 7. 实测结论

“全字段新增验证”已确认以下事实：

- 所有目标字段可提交并成功落库
- 回拉导出可完整读到对应字段值
- 该结构可作为复刻系统的最终字段标准
